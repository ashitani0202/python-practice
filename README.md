# python-practice
github and python practice repositories<br>
Possibility to use programming languages other than python<br>

githubとpythonの練習用のリポジトリ<br>
python以外のプログラム言語をあげる可能性あり<br>

## commit rules<br>

### 1. Add a prefix<br>
add: Add new (file) function<br>
update: Function fix (not a bug)<br>
feat: New features<br>
fix: Bug fix<br>
docs: Documentation only changes<br>
style: spaces, formatting, adding semicolons, etc.<br>
refactor: Code improvement (refactor) that does not affect the specifications<br>
perf: Performance improvement related<br>
test: Test related<br>
chore: Build, auxiliary tools, library related<br>
remove: Delete (file)<br>

### 2. Write your reasons<br>
feat: 〇〇なため、△△を追加<br>
chore: added hoge. Instead of chore:, add hoge for network communication. It is easier to understand if you write<br>

## commitルール<br>

### 1. Prefixをつける<br>
add：新規（ファイル）機能追加<br>
update：機能修正（バグではない）<br>
feat: 新しい機能<br>
fix: バグの修正<br>
docs: ドキュメントのみの変更<br>
style: 空白、フォーマット、セミコロン追加など<br>
refactor: 仕様に影響がないコード改善(リファクタ)<br>
perf: パフォーマンス向上関連<br>
test: テスト関連<br>
chore: ビルド、補助ツール、ライブラリ関連<br>
remove：削除（ファイル）<br>

### 2. 理由を書く<br>
chore: hogeを追加ではなく<br>
chore: ネットワーク通信するため、hogeを追加<br>
と書いた方がよりわかりやすい<br>
